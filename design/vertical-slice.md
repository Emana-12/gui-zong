## 什么是这个 Vertical Slice？

一个完整可玩的薄片——从开始到结束走完核心循环。用最少的内容证明"这个游戏好玩"。

**完整周期**：进入竞技场 → 用三式剑招战斗 → 连击积累 → 触发万剑归宗 → 击杀或死亡 → 重新开始

## 敌人设计：近战士兵

唯一敌人类型，需要替代原型中的静态假人以验证"战斗中的实时决策"。

### 行为状态机

```
IDLE → CHASE → ATTACK → RETREAT → IDLE
```

| 状态 | 行为 | 持续时间 |
|------|------|----------|
| IDLE | 原地等待，检测玩家进入 8m 范围 | 直到检测到 |
| CHASE | 向玩家移动，速度 3.0 m/s | 直到距离 < 2.0m |
| ATTACK | 执行攻击动画，前方 2.0m 扇形判定 | 0.5s |
| RETREAT | 后退 3.0m，速度 4.0 m/s（闪避） | 0.5s |
| HIT_STUN | 受击硬直，中断当前状态 | 0.3s |

### 属性

- 生命值：5
- 攻击伤害：1
- 受击变色：白→红（HP 递减）

### 设计意图

- **追逐行为**迫使玩家移动，不能站在原地输出
- **攻击行为**让绕剑式的"化解"有了用武之地
- **后退行为**防止玩家无限连击——敌人会逃跑
- 5 HP 要求至少 5 次游剑式或 2 次钻剑式，鼓励三式切换

## 方向性命中判定（取代自动锁定）

### 原型问题

原型使用自动锁定最近敌人——玩家不需要瞄准，失去了空间决策。

### 生产实现

1. **检测方向**：玩家面朝方向（forward vector）
2. **检测形状**：前方 90° 扇形，半径 2.5m
3. **判定逻辑**：
   - 执行期间每帧检测
   - 在扇形范围内找到最近敌人
   - 命中该敌人并结束（每招只命中 1 个）
4. **多个敌人在扇形内**：命中最靠近面朝方向的那个（点积最大）

### 手感影响

- 玩家需要**面向**敌人——产生空间决策
- 面对多个敌人时需要选择目标——增加战术深度
- 误判方向 = miss——精确即力量的直接体现

## 流光轨迹（基础实现）

### 绕剑式：墨色流光

- 颜色：深墨色 (#1A1A2E)，半透明
- 形状：跟随剑尖运动的带状网格
- 宽度：0.05m
- 淡出：执行结束后 0.3s 内渐隐

### 游剑式/钻剑式：金色剑气

- 颜色：金色 (#C9A96E)，带微发光
- 形状：与绕剑式相同的带状网格
- 游剑式：窄（0.03m）、短尾迹
- 钻剑式：宽（0.08m）、长尾迹

### 实现方式

使用 MeshTrail3D 或自定义 Line3D 节点。每帧记录剑尖位置，生成网格。不需要 GPU 粒子——基础网格即可验证"流光感"。

## HUD 设计

### 元素布局

```
┌─────────────────────────────────────────┐
│  [墨滴×N] 生命值          连击: 23       │
│                                         │
│                                         │
│                                         │
│                                         │
│                              万剑归宗    │
│                              [████░░]    │
│                                         │
│           ● (玩家)                       │
│                                         │
└─────────────────────────────────────────┘
```

### 墨滴生命值

- 每个墨滴 = 1 HP
- 满血：5 个完整墨滴
- 受击：1 个墨滴碎裂/消失
- 位置：左上角

### 连击计数

- 大号数字，右上角
- 不同剑式命中时数字跳动变大
- 3 秒超时后渐隐消失

### 万剑归宗充能条

- 右下角，竖条
- 0% 时不可见
- ≥ 10 连击后显示"SPACE 可用"
- 20 连击自动触发时闪烁

## 验收标准

垂直切片通过需满足以下**全部**条件：

### 功能验收

1. [ ] 玩家可以使用 3 种剑式战斗，每式有明确的手感差异
2. [ ] 方向性命中判定工作——需要面向敌人才能命中
3. [ ] 近战士兵敌人可以追击、攻击和后退
4. [ ] 连击系统积累正确——不同剑式计数，同剑式不断连
5. [ ] 万剑归宗可手动触发（≥10 连击），对范围内敌人造成伤害
6. [ ] 基础流光轨迹跟随剑尖运动，绕/游/钻有颜色区分
7. [ ] 顿帧 + 屏幕震动在命中时触发
8. [ ] HUD 正确显示生命值、连击计数、万剑归宗充能
9. [ ] 死亡后可重新开始，无卡死

### 手感验收（需实机测试）✨ CD-PLAYTEST BLOCKING

10. [ ] 至少 1 名非开发者试玩后能自然使用至少 2 种剑式
11. [ ] 试玩者在 5 分钟内能触发至少 1 次万剑归宗
12. [ ] 试玩者报告"想再来一次"而非"终于结束了"

### 性能验收

13. [ ] Web 导出 ≥ 50fps（目标 60fps）
14. [ ] 无内存泄漏（10 分钟内不增长 >20MB）
