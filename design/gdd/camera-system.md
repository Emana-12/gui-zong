# 摄像机系统 (Camera System)

> **Status**: Designed
> **Author**: User + Agents
> **Last Updated**: 2026-04-21
> **Implements Pillar**: 极简即美学 (Pillar 2)

## Overview

**摄像机系统**控制玩家在竞技场中的视角——它跟随玩家角色、框定战斗区域、在万剑归宗时提供视觉冲击、在死亡时配合画面墨化。摄像机是玩家"看"这个水墨世界的窗口。

**职责边界：**
- **做**：摄像机跟随（平滑追踪玩家位置）、视角框定（保持竞技场可见）、特殊效果（万剑归宗的镜头拉远、死亡的画面静止）
- **不做**：输入处理（不直接读取输入）、画面特效（由后处理 pass 负责）

**为什么需要它：** 3D 动作游戏的摄像机直接影响玩家的战斗感知。摄像机太近=看不清敌人，太远=失去代入感。这个系统确保玩家在战斗中始终看到足够的信息来做精确决策。

## Player Fantasy

**玩家不会意识到摄像机的存在——但他们始终能看到该看到的一切。**

玩家感受到的是：
- 战斗中，敌人始终在视野内——不需要手动转动视角
- 闪避时，画面轻微跟随但不过度晃动——稳定感
- 万剑归宗时，镜头缓缓拉远——壮观感来自"看到更多"而非"特效更多"
- 死亡时，画面静止——墨化动画从稳定的视角开始

**情感目标：** 稳定感。玩家应该感觉摄像机是自己的第三只眼——它从不干扰战斗，从不错过关键信息。

**支柱对齐：** 服务于"极简即美学"(Pillar 2)——摄像机的极简就是"不做多余动作"。好的摄像机是看不见的摄像机。

## Detailed Design

### Core Rules

1. 摄像机是一个固定角度的 3/4 俯视角（约 45° 向下倾斜），不做自由旋转——玩家不需要控制摄像机
2. 摄像机平滑跟随玩家角色的 XZ 平面位置，Y 轴高度固定
3. 跟随使用线性插值（lerp）实现平滑移动，避免画面抖动
4. 摄像机始终框定竞技场的中心区域——确保玩家不会因为摄像机跟随而看不到竞技场边缘的敌人
5. 万剑归宗触发时，摄像机缓缓拉远（FOV 增大或距离增大），展示全景
6. 死亡时，摄像机完全冻结（停止跟随），配合画面墨化效果

### Camera Configuration

| 参数 | 值 | 说明 |
|------|-----|------|
| 角度 | 45° 向下倾斜 | 3/4 俯视角，兼顾角色和环境可见度 |
| 距离 | 8.0 m（从玩家到摄像机） | 标准战斗距离 |
| FOV | 60° | 标准 FOV，不做鱼眼效果 |
| 跟随速度 | lerp_factor = 5.0 | 跟随平滑度（越大越灵敏） |
| 高度固定 | 6.0 m（Y 轴） | 摄像机高度不变 |

### Camera Behaviors by Game State

| 游戏状态 | 摄像机行为 | 说明 |
|---------|---------|------|
| TITLE | 固定位置，缓慢旋转 | 标题画面的氛围镜头 |
| COMBAT | 平滑跟随玩家 | 标准战斗跟随 |
| INTERMISSION | 轻微拉远，缓慢旋转 | 间歇时的环境展示 |
| DEATH | 完全冻结 | 配合墨化效果 |
| RESTART | 瞬切回 COMBAT 位置 | 快速重置 |

### Camera Effects

| 效果 | 触发条件 | 行为 | 时长 |
|------|---------|------|------|
| 万剑归宗拉远 | `myriad_swords_triggered` | FOV 从 60° 增大到 75°，或距离从 8m 增大到 12m | 2 秒蓄力，1 秒恢复 |
| 受击震动 | `take_damage` | 轻微水平抖动（±0.1m） | 0.1 秒 |
| 命中顿帧 | 命中事件 | 摄像机暂停 2 帧 | 2 帧 |

### Interactions with Other Systems

**对外发出的接口：**

| 接口 | 类型 | 说明 | 接收系统 |
|------|------|------|---------|
| `get_camera_position()` | 查询 | 摄像机当前世界坐标 | HUD/UI（世界坐标转屏幕坐标） |
| `get_camera_forward()` | 查询 | 摄像机前方向向量 | 命中反馈（屏幕震动方向） |
| `trigger_effect(effect_name: String)` | 触发 | 触发摄像机效果 | 连击/万剑归宗, 命中反馈 |

**接收的外部触发：**

| 触发来源 | 事件 | 响应 |
|---------|------|------|
| 玩家控制器 | `get_position()` | 平滑跟随玩家位置 |
| 游戏状态管理 | `state_changed` | 根据状态切换摄像机行为 |
| 连击/万剑归宗 | 万剑归宗触发 | 触发拉远效果 |
| 命中反馈 | 受击事件 | 触发震动/顿帧效果 |

### Public API

| 方法 | 签名 | 返回值 | 说明 |
|------|------|--------|------|
| `get_camera_position` | `get_camera_position()` | `Vector3` | 摄像机当前世界坐标 |
| `get_camera_forward` | `get_camera_forward()` | `Vector3` | 摄像机前方向向量 |
| `trigger_effect` | `trigger_effect(effect_name: String)` | `void` | 触发摄像机效果（震动/拉远/顿帧） |
| `set_follow_target` | `set_follow_target(target: Node3D)` | `void` | 设置跟随目标（默认为玩家） |
| `set_follow_speed` | `set_follow_speed(speed: float)` | `void` | 设置跟随平滑度 |

## Formulas

**摄像机跟随插值：**

`camera_pos = lerp(camera_pos, player_pos, follow_speed * delta)`

**变量：**

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| 当前位置 | `camera_pos` | Vector3 | — | 摄像机当前世界坐标 |
| 目标位置 | `player_pos` | Vector3 | — | 玩家当前世界坐标（仅 XZ，Y 固定） |
| 跟随速度 | `follow_speed` | float | 3.0–10.0 | lerp 系数（越大越灵敏） |
| 帧时间 | `delta` | float | ~0.016s @60fps | 上一帧的时间 |

**输出：** 平滑移动的摄像机位置。`follow_speed=5.0` 时，每帧移动目标距离的约 8%。

**FOV 拉远插值（万剑归宗）：**

`current_fov = lerp(current_fov, target_fov, fov_lerp_speed * delta)`

**变量：**

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| 当前 FOV | `current_fov` | float | 60–75° | 当前视野角度 |
| 目标 FOV | `target_fov` | float | 60–75° | 目标视野角度 |
| FOV 速度 | `fov_lerp_speed` | float | 2.0–5.0 | FOV 变化速度 |

## Edge Cases

- **如果玩家快速来回移动（如闪避后立即反方向闪避）**：摄像机平滑跟随不过度反应——`follow_speed` 限制了最大跟随速率。
- **如果竞技场边缘的敌人不在摄像机视野内**：固定 3/4 俯视角 + 8m 距离应覆盖竞技场大部分区域。如果仍有死角，需要调整竞技场大小或摄像机距离。
- **如果万剑归宗效果被中断**（如玩家死亡）：FOV 立即重置到 60°，不等待插值完成。
- **如果受击震动在万剑归宗拉远期间触发**：震动叠加在拉远效果上，两者独立计算。
- **如果多个摄像机效果同时触发**（震动+顿帧）：按优先级顺序执行——顿帧（2帧）优先于震动。

| 系统 | 依赖类型 | 接口 |
|------|---------|------|
| 玩家控制器 | 硬依赖 | `get_position()` — 摄像机跟随目标 |

| 系统 | 依赖类型 | 接口 |

## Tuning Knobs

| 旋钮 | 默认值 | 安全范围 | 说明 |
|------|--------|---------|------|
| `camera_distance` | 8.0 m | 5.0–12.0 m | 摄像机到玩家的距离 |
| `camera_height` | 6.0 m | 4.0–10.0 m | 摄像机 Y 轴固定高度 |
| `camera_angle` | 45° | 30–60° | 摄像机向下倾斜角度 |
| `follow_speed` | 5.0 | 3.0–10.0 | 跟随平滑度 |
| `fov_normal` | 60° | 50–70° | 标准 FOV |
| `fov_wanjian` | 75° | 65–90° | 万剑归宗时的 FOV |
| `shake_intensity` | 0.1 m | 0.05–0.3 m | 受击震动幅度 |
| `hit_stop_frames` | 2 | 1–4 | 命中顿帧帧数 |

## Visual/Audio Requirements

**本系统不直接产生视觉效果——它控制视角。** 摄像机效果（拉远/震动/顿帧）的视觉表现由下游系统负责：
- 万剑归宗拉远：摄像机 FOV 变化（本系统负责）+ 金色流光视觉效果（流光轨迹系统负责）
- 受击震动：摄像机位置抖动（本系统负责）+ 屏幕边缘墨迹侵蚀（HUD 系统负责）
- 命中顿帧：全局暂停 2 帧（本系统负责）+ 命中视觉效果（命中反馈负责）

## UI Requirements

**HUD 坐标转换：** HUD/UI 系统需要将 3D 世界坐标转换为 2D 屏幕坐标（如敌人血条位置、伤害数字弹出位置）。使用 `get_camera_position()` 和 Godot 的 `Camera3D.unproject_position()` 实现。

## Acceptance Criteria

- **GIVEN** 玩家从位置 A 移动到位置 B，**WHEN** 摄像机跟随，**THEN** 摄像机在 0.5 秒内平滑到达玩家附近位置
- **GIVEN** 战斗中，**WHEN** 摄像机跟随玩家，**THEN** 所有敌人始终在视野内（竞技场大小需匹配摄像机距离）
- **GIVEN** 万剑归宗触发，**WHEN** FOV 从 60° 变化到 75°，**THEN** 在 2 秒内平滑过渡到目标 FOV
- **GIVEN** 万剑归宗结束，**WHEN** FOV 恢复，**THEN** 在 1 秒内平滑恢复到 60°
- **GIVEN** 玩家受击，**WHEN** `trigger_effect("shake")` 被调用，**THEN** 摄像机在 0.1 秒内水平抖动 ±0.1m
- **GIVEN** 命中事件触发，**WHEN** 命中顿帧生效，**THEN** 摄像机暂停 2 帧后恢复正常跟随
- **GIVEN** 游戏状态为 DEATH，**WHEN** 玩家死亡，**THEN** 摄像机完全冻结，位置不变
- **GIVEN** 游戏状态为 TITLE，**WHEN** 标题画面显示，**THEN** 摄像机在固定位置缓慢旋转

## Open Questions

- 万剑归宗的镜头效果是拉远 FOV 还是增大距离？FOV 变化更自然但可能产生鱼眼畸变；距离增大更可控但可能穿帮（看到竞技场外的虚空）。
- 摄像机是否需要跟随敌人位置（如面向最近敌人的方向）？当前设计只跟随玩家。如果需要面向敌人，需要额外的旋转逻辑。
- 竞技场大小和摄像机距离需要在原型阶段联合测试——确保两者匹配时所有敌人可见。
