# 连击/万剑归宗系统 (Combo/Myriad Swords)

> **Status**: Designed
> **Author**: User + Agents
> **Last Updated**: 2026-04-21
> **Implements Pillar**: 万剑归宗即高潮 (Pillar 5), 三式皆平等 (Pillar 4)

## Overview

**连击/万剑归宗系统**是《归宗》的高潮引擎——通过三式切换积累连击，达到阈值时触发万剑归宗终极场景。万剑归宗是玩家技巧的最高表达——不靠运气，只靠技巧。

**职责边界：**
- **做**：连击计数（三式切换）、连击阈值管理、万剑归宗触发、万剑归宗效果协调
- **不做**：剑招执行（三式剑招系统）、视觉效果（流光轨迹系统）、伤害计算（命中判定层）

## Player Fantasy

**连击越高，万剑归宗越壮观——你的技巧就是你的力量。**

玩家感受到的是：
- 连续使用不同剑式击中敌人时，连击计数器在增长——墨点在积累
- 连击达到阈值时，万剑归宗蓄力环填满金色——"来了！"
- 万剑归宗触发时，万道金色轨迹同时飞舞——所有积蓄的力量在一瞬间释放
- 连击越高，万剑归宗的轨迹越多、范围越大、伤害越高——技巧越高，高潮越壮观

**支柱对齐：**
- Pillar 5（万剑归宗即高潮）：万剑归宗不靠运气触发，只靠技巧触发
- Pillar 4（三式皆平等）：连击需要三式切换——只用一式无法积累连击

## Detailed Design

### Core Rules

1. 连击系统监听三式剑招系统的 `form_activated` 信号和命中判定层的 `hit_landed` 信号
2. 每次有效命中增加连击计数——但只有**不同剑式的连续命中**才计入连击
3. 连击断开条件：被敌人击中、超过连击超时时间（无命中）、使用同一剑式连续命中（不增加但不断连）
4. 连击达到阈值（默认 10 次不同剑式命中）时，万剑归宗蓄力完成
5. 万剑归宗可由玩家主动触发（按下指定键）或自动触发（连击达到 2 倍阈值）
6. 万剑归宗效果：对范围内所有敌人造成伤害，轨迹数量和范围随连击数增长

### Combo Rules

| 规则 | 说明 |
|------|------|
| 连击计数 | 不同剑式连续命中才+1（游→钻→绕→游=4 连击） |
| 同式连续 | 不增加连击但不断连（游→游→钻=2 连击） |
| 被击中 | 连击归零 |
| 超时 | 连击超时时间（默认 3 秒）内无命中则连击归零 |

### Myriad Swords Rules

| 参数 | 值 | 说明 |
|------|-----|------|
| 蓄力阈值 | 10 连击 | 需要 10 次不同剑式命中 |
| 触发方式 | 手动（按指定键）或自动（20 连击） | 手动=战术选择，自动=极致技巧 |
| 轨迹数量 | 基础 20 + 连击数 × 2 | 10 连击=40 条，20 连击=60 条 |
| 伤害 | 基础 5 + 连击数 × 0.5 | 10 连击=10 伤害，20 连击=15 伤害 |
| 范围 | 基础 5m + 连击数 × 0.3m | 10 连击=8m，20 连击=11m |
| 冷却 | 10 秒 | 触发后需要重新积累连击 |

### Interactions with Other Systems

**对外发出的接口：**

| 接口 | 类型 | 说明 | 接收系统 |
|------|------|------|---------|
| `get_combo_count()` | 查询 | 当前连击数 | HUD, 计分 |
| `get_charge_progress()` | 查询 | 万剑归宗蓄力进度（0–1） | HUD（蓄力环） |
| `is_myriad_ready()` | 查询 | 万剑归宗是否就绪 | HUD |
| `combo_changed` | 信号 | 连击数变化 | HUD（墨点计数器） |
| `myriad_triggered` | 信号 | 万剑归宗触发 | 流光轨迹, 命中反馈, 摄像机, 音频 |
| `myriad_damage` | 信号 | 万剑归宗造成范围伤害 | 命中判定层, 敌人系统 |

**接收的外部触发：**

| 触发来源 | 事件 | 响应 |
|---------|------|------|
| 命中判定层 | `hit_landed` | 增加连击计数 |
| 三式剑招系统 | `form_activated` | 记录当前剑式 |
| 玩家控制器 | `player_died` / `take_damage` | 连击归零 |
| 输入系统 | 万剑归宗触发键 | 手动触发万剑归宗 |
| 游戏状态管理 | `state_changed` | 非 COMBAT 状态时冻结连击 |

### Public API

| 方法 | 签名 | 返回值 | 说明 |
|------|------|--------|------|
| `get_combo_count` | `get_combo_count()` | `int` | 当前连击数 |
| `get_charge_progress` | `get_charge_progress()` | `float` | 蓄力进度 0–1 |
| `is_myriad_ready` | `is_myriad_ready()` | `bool` | 是否可以触发万剑归宗 |
| `trigger_myriad` | `trigger_myriad()` | `bool` | 手动触发万剑归宗（成功返回 true） |
| `reset_combo` | `reset_combo()` | `void` | 重置连击（被击中/死亡时调用） |

## Formulas

**万剑归宗轨迹数：**

`myriad_trail_count = base_trails + combo_count * trails_per_combo`

**变量：**

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| 基础轨迹数 | `base_trails` | int | 20 | 最少轨迹数 |
| 连击数 | `combo_count` | int | 10–∞ | 当前连击数 |
| 每连击轨迹增量 | `trails_per_combo` | int | 2 | 每次连击增加的轨迹数 |

**输出：** 40–∞ 条轨迹。上限由流光轨迹系统的 `max_concurrent_trails` 约束（默认 50）。

**万剑归宗伤害：**

`myriad_damage = base_damage + combo_count * damage_per_combo`

**示例：** 10 连击 → 5 + 10×0.5 = 10 伤害。20 连击 → 5 + 20×0.5 = 15 伤害。

## Edge Cases

- **如果万剑归宗蓄力完成后玩家被击中**：连击归零，蓄力重置——需要重新积累。
- **如果万剑归宗触发时场上没有敌人**：仍然触发（视觉效果），但不造成伤害。
- **如果连击超时恰好与万剑归宗触发同时发生**：优先处理万剑归宗触发（已蓄力）。
- **如果自动触发（20 连击）和手动触发同时发生**：只触发一次，不重复。

## Tuning Knobs

| 旋钮 | 默认值 | 安全范围 | 说明 |
|------|--------|---------|------|
| `combo_timeout` | 3.0s | 1.0–5.0s | 连击超时时间。太短=连击难维持，太长=连击无压力 |
| `charge_threshold` | 10 | 5–20 | 万剑归宗蓄力阈值 |
| `auto_trigger_threshold` | 20 | 15–30 | 自动触发阈值（2 倍蓄力阈值） |
| `myriad_cooldown` | 10.0s | 5.0–20.0s | 万剑归宗冷却时间 |
| `trails_per_combo` | 2 | 1–5 | 每连击增加的轨迹数 |
| `damage_per_combo` | 0.5 | 0.2–1.0 | 每连击增加的万剑归宗伤害 |
| `range_per_combo` | 0.3m | 0.1–0.5m | 每连击增加的万剑归宗范围 |

## Visual/Audio Requirements

**视觉需求：**
- 连击计数器：墨点积累（HUD 系统负责）
- 蓄力环：墨色圆环逐渐填满金色（HUD 系统负责）
- 万剑归宗：万道金色轨迹同时飞舞（流光轨迹系统负责）
- 摄像机拉远（摄像机系统负责）

**音频需求：**
- 连击音效：每次连击+1 时的"叮"声
- 蓄力完成音效：蓄力环填满时的提示音
- 万剑归宗音效：渐强爆发

## UI Requirements

**HUD 连击计数器：** 墨点积累 + 金色数字。监听 `combo_changed` 信号。
**HUD 蓄力环：** 墨色圆环填满金色。监听 `get_charge_progress()`。

## Acceptance Criteria

- **GIVEN** 游→钻→绕连续命中，**WHEN** 第 3 次命中，**THEN** 连击数=3
- **GIVEN** 连击数=3，**WHEN** 再次使用游剑式命中（与上一式相同），**THEN** 连击数不变（仍为 3）
- **GIVEN** 连击数=10，**WHEN** `get_charge_progress()` 被查询，**THEN** 返回 1.0（蓄力完成）
- **GIVEN** 连击数=10 且蓄力完成，**WHEN** `trigger_myriad()` 被调用，**THEN** 万剑归宗触发，`myriad_triggered` 信号发出
- **GIVEN** 连击数=20，**WHEN** 达到自动触发阈值，**THEN** 万剑归宗自动触发
- **GIVEN** 连击数=5，**WHEN** 被敌人击中，**THEN** 连击归零
- **GIVEN** 连击数=7，**WHEN** 3 秒内无命中，**THEN** 连击归零（超时）
- **GIVEN** 万剑归宗触发，**WHEN** 10 连击，**THEN** 轨迹数=40，伤害=10，范围=8m
- **GIVEN** 万剑归宗冷却中，**WHEN** `trigger_myriad()` 被调用，**THEN** 返回 false

## Open Questions

- 连击是否应该奖励"混合连击"（三种剑式各用一次后+额外分数）？当前设计只计不同剑式。
- 万剑归宗的伤害是否应该对每个敌人独立计算？（考虑敌人的防御值）
- 蓄力阈值（10）是否太容易达到？需要在原型阶段测试实际连击维持难度。
