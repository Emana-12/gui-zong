# 玩家控制器 (Player Controller)

> **Status**: Designed
> **Author**: User + Agents
> **Last Updated**: 2026-04-21
> **Implements Pillar**: 精确即力量 (Pillar 1)

## Overview

**玩家控制器**是玩家在游戏世界中的化身——它管理玩家角色的移动、闪避、生命值和死亡。所有"玩家做了什么"的行为都通过这个系统表达。

**职责边界：**
- **做**：3D 空间移动（前进/后退/左右）、闪避（带无敌帧）、生命值管理（受伤/恢复/归零）、死亡触发（通知游戏状态管理）
- **不做**：剑招逻辑（由三式剑招系统处理）、输入映射（由输入系统处理）、视觉效果（由流光轨迹/命中反馈处理）

**为什么需要它：** 玩家控制器是"精确即力量"的操作基础——移动的精确性（闪避的时机窗口）、生命值的精确管理（被命中即惩罚）、死亡的精确触发（没有 HP 侥幸）。

## Player Fantasy

**玩家感觉自己就是那个持剑者——不是操控一个角色，是成为他。**

玩家感受到的是：
- 按下方向键时，角色如行云流水般移动——没有惯性的拖沓，没有加速的迟钝
- 按下闪避键时，角色瞬间闪出攻击范围——无敌帧的时机窗口精确到帧
- 被敌人击中时，墨色吞噬金色——生命值的视觉衰减让每一次受伤都感到代价
- 生命值归零时，角色轮廓溶解于墨色——死亡不是爆炸，是融入山水

**情感目标：** 控制感。玩家应该感觉自己的每一个输入都精确地转化为角色的动作——没有延迟、没有偏差、没有"我没按这个"。

**支柱对齐：** 服务于"精确即力量"(Pillar 1)——闪避的无敌帧是精确打击的防守面。服务"三式皆平等"(Pillar 4)——移动系统不偏向任何一式，三式在相同的移动基础上使用。

## Detailed Design

### Core Rules

1. 玩家角色是一个 CharacterBody3D 节点，使用 Godot 的内置移动系统（`move_and_slide()`）
2. 移动方向由输入系统的 `get_move_direction()` 驱动——WASD/摇杆输入转化为 3D 空间方向向量
3. 移动速度恒定（无加速/减速曲线）——玩家按下方向键时立即以最大速度移动，松开时立即停止
4. 闪避通过 `dodge` 动作触发，产生一次快速位移（向当前移动方向，或默认向前）
5. 闪避有无敌帧窗口（从闪避开始后 N 帧内不受伤害），闪避结束后有冷却时间
6. 生命值系统：玩家有固定最大生命值（默认 3），每次受击扣 1 点。生命值归零时触发死亡
7. 死亡时发出 `player_died` 信号，通知游戏状态管理切换到 DEATH 状态
8. 玩家角色始终面向敌人方向（自动朝向最近的敌人），不手动控制朝向

### Movement Rules

| 参数 | 值 | 说明 |
|------|-----|------|
| 移动速度 | 5.0 m/s | 恒定速度，无加速曲线 |
| 移动模式 | 即时响应 | 按下=全速，松开=立即停止 |
| 朝向模式 | 自动朝向 | 面向最近的敌人 |
| 空间 | 3D 平面（XZ） | Y 轴受重力影响，不做跳跃 |

### Dodge Rules

| 参数 | 值 | 说明 |
|------|-----|------|
| 闪避距离 | 3.0 m | 单次闪避的位移距离 |
| 闪避时长 | 0.2 秒 | 闪避位移的持续时间 |
| 无敌帧 | 前 0.15 秒 | 闪避开始后 0.15 秒内无敌 |
| 冷却时间 | 0.5 秒 | 闪避结束后不能立即再次闪避 |
| 闪避方向 | 当前移动方向 | 无移动输入时默认向前 |

### Health Rules

| 参数 | 值 | 说明 |
|------|-----|------|
| 最大生命值 | 3 | 固定值，不可升级 |
| 受击扣血 | 1 点/次 | 每次被命中扣 1 点 |
| 无敌时间（受击后） | 0.5 秒 | 受击后短暂无敌，防止连续受击 |
| 恢复 | 无 | 不自动恢复，不掉落恢复道具 |
| 死亡触发 | 生命值 = 0 | 发出 `player_died` 信号 |

### States and Transitions

| 状态 | 进入条件 | 退出条件 | 行为 |
|------|---------|---------|------|
| **IDLE** | 无移动输入 | 收到移动输入 | 静止，自动朝向敌人 |
| **MOVING** | 收到移动输入 | 移动输入消失 | 以恒定速度移动 |
| **DODGING** | `dodge` 动作触发 | 闪避结束（0.2 秒） | 快速位移，无敌帧 |
| **DODGE_COOLDOWN** | 闪避结束 | 冷却结束（0.5 秒） | 可移动但不能再次闪避 |
| **HIT_STUN** | 受击 | 无敌时间结束（0.5 秒） | 可受击但有无敌帧保护 |
| **DEAD** | 生命值归零 | 游戏重启 | 不可操作，角色溶解 |

### Interactions with Other Systems

**对外发出的接口：**

| 接口 | 类型 | 说明 | 接收系统 |
|------|------|------|---------|
| `get_position()` | 查询 | 玩家当前世界坐标 | 摄像机, 敌人 AI, 碰撞检测 |
| `get_velocity()` | 查询 | 玩家当前速度向量 | 摄像机, 动画系统 |
| `get_health()` | 查询 | 当前生命值 | HUD/UI, 敌人系统 |
| `get_max_health()` | 查询 | 最大生命值 | HUD/UI |
| `is_invincible()` | 查询 | 是否处于无敌状态 | 命中判定层 |
| `is_dodging()` | 查询 | 是否正在闪避 | 命中判定层, 动画系统 |
| `take_damage(amount: int)` | 设置 | 受到伤害 | 命中判定层 |
| `player_died` | 信号 | 玩家死亡 | 游戏状态管理, 音频, HUD |
| `health_changed` | 信号 | 生命值变化 | HUD/UI |

**接收的外部触发：**

| 触发来源 | 事件 | 响应 |
|---------|------|------|
| 输入系统 | `get_move_direction()` | 更新移动方向 |
| 输入系统 | `is_action_just_pressed("dodge")` | 触发闪避 |
| 命中判定层 | 敌人攻击命中 | 调用 `take_damage(1)` |
| 游戏状态管理 | `state_changed` | COMBAT 状态下可操作，其他状态冻结 |

### Public API

| 方法 | 签名 | 返回值 | 说明 |
|------|------|--------|------|
| `get_position` | `get_position()` | `Vector3` | 玩家当前世界坐标 |
| `get_velocity` | `get_velocity()` | `Vector3` | 玩家当前速度向量 |
| `get_health` | `get_health()` | `int` | 当前生命值 |
| `get_max_health` | `get_max_health()` | `int` | 最大生命值 |
| `is_invincible` | `is_invincible()` | `bool` | 是否处于无敌状态（闪避/受击后） |
| `is_dodging` | `is_dodging()` | `bool` | 是否正在闪避 |
| `take_damage` | `take_damage(amount: int)` | `void` | 受到伤害。生命值归零时触发死亡 |
| `heal` | `heal(amount: int)` | `void` | 恢复生命值（不超过最大值） |

## Formulas

**闪避无敌帧计算：**

`invincible_frames = ceil(dodge_invincible_duration * target_fps)`

**变量：**

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| 无敌时长 | `dodge_invincible_duration` | float | 0.1–0.2 秒 | 闪避无敌帧持续时间 |
| 目标帧率 | `target_fps` | float | 30–60 | 游戏目标帧率 |

**输出：** 6 帧（0.15 秒 @40fps）到 12 帧（0.2 秒 @60fps）

**受击无敌帧计算：**

`hit_stun_frames = ceil(hit_stun_duration * target_fps)`

**输出：** 15 帧（0.5 秒 @30fps）到 30 帧（0.5 秒 @60fps）

## Edge Cases

- **如果闪避触发时玩家没有移动输入**：默认向前闪避（玩家角色朝向方向）。
- **如果闪避触发时正在闪避冷却中**：忽略输入，不缓冲（闪避不进入输入缓冲）。
- **如果玩家在无敌帧期间被多个敌人同时命中**：只处理第一次命中，后续命中被无敌状态吸收。
- **如果生命值已经为 0 时再次受到伤害**：忽略，不触发多次死亡信号。
- **如果 `heal` 在生命值已满时调用**：忽略。
- **如果游戏状态不是 COMBAT 时收到移动输入**：忽略，角色冻结。
- **如果自动朝向的敌人被击杀**：在下一帧重新寻找最近的敌人朝向。

## Tuning Knobs

| 旋钮 | 默认值 | 安全范围 | 说明 |
|------|--------|---------|------|
| `move_speed` | 5.0 m/s | 3.0–8.0 m/s | 移动速度。太快=难以精确控制，太慢=战斗节奏拖沓 |
| `dodge_distance` | 3.0 m | 1.5–5.0 m | 闪避位移距离 |
| `dodge_duration` | 0.2 秒 | 0.1–0.4 秒 | 闪避位移时长 |
| `dodge_invincible_duration` | 0.15 秒 | 0.1–0.2 秒 | 闪避无敌帧时长 |
| `dodge_cooldown` | 0.5 秒 | 0.3–1.0 秒 | 闪避冷却时间 |
| `hit_stun_duration` | 0.5 秒 | 0.3–1.0 秒 | 受击后无敌时间 |
| `max_health` | 3 | 1–5 | 最大生命值 |

## Visual/Audio Requirements

**视觉需求（由下游系统负责）：**
- 移动时角色动画：水墨风格化的行走/跑步循环（非匀速——出招快，收招慢）
- 闪避时：角色轮廓模糊化（水墨拖影效果）
- 受击时：屏幕边缘墨迹侵蚀动画（Art Bible Section 7）
- 死亡时：角色轮廓渐变为淡墨，融入背景山水（Art Bible Section 2）
- 生命值：墨滴消耗的视觉反馈（HUD 系统负责）

**音频需求：**
- 闪避音效：快速的"嗖"声
- 受击音效：闷响
- 死亡音效：余音消散

## UI Requirements

**HUD 生命值显示：** 墨滴消耗式——满血=饱满墨滴，低血=干涸墨迹。由 HUD/UI 系统监听 `health_changed` 信号实现。

## Acceptance Criteria

- **GIVEN** 玩家按下 W 键，**WHEN** 游戏状态为 COMBAT，**THEN** 玩家以 `move_speed` 向前移动
- **GIVEN** 玩家释放所有方向键，**WHEN** 正在移动中，**THEN** 玩家立即停止（无惯性）
- **GIVEN** 玩家按下闪避键，**WHEN** 不在闪避冷却中，**THEN** 玩家向当前移动方向闪避 `dodge_distance` 距离
- **GIVEN** 玩家正在闪避的前 0.15 秒内，**WHEN** `is_invincible()` 被查询，**THEN** 返回 true
- **GIVEN** 玩家闪避结束后 0.5 秒内，**WHEN** 再次按下闪避键，**THEN** 输入被忽略
- **GIVEN** 玩家生命值 = 3，**WHEN** `take_damage(1)` 被调用，**THEN** 生命值变为 2，发出 `health_changed` 信号
- **GIVEN** 玩家生命值 = 1，**WHEN** `take_damage(1)` 被调用，**THEN** 生命值变为 0，发出 `player_died` 信号
- **GIVEN** 玩家生命值 = 0，**WHEN** `take_damage(1)` 被调用，**THEN** 忽略，不触发多次死亡
- **GIVEN** 游戏状态为 TITLE，**WHEN** 玩家按下方向键，**THEN** 角色不移动
- **GIVEN** 玩家无移动输入，**WHEN** 最近敌人被击杀，**THEN** 下一帧重新朝向新的最近敌人

## Open Questions

- 自动朝向最近敌人的逻辑是否需要平滑旋转（Slerp）？当前设计假设瞬时朝向。平滑旋转更自然但可能影响精确打击时的朝向响应。
- 生命值最大值（3）是否需要成为 Tuning Knob？对于 Demo 阶段可以固定，但完整游戏可能需要调整难度。
- 闪避是否应该有方向限制（只能向前/向侧方，不能向后）？当前设计允许任何方向。向后闪避可能过于保守。
