# 死亡画面 UX 规范

> **Epic**: HUD/UI 系统
> **Created**: 2026-04-23
> **Sprint**: S03-07
> **Status**: Draft

## Overview

死亡画面在玩家生命值归零、游戏状态切换到 DEATH 时显示。设计目标是在保持水墨美学的同时，清晰传达"游戏结束"信息并引导玩家重新开始。

## Player Fantasy

玩家在激烈战斗后突然死亡，画面从墨色中浮现"剑已断"三个白色大字，配合金墨色分数显示——既是一次失败的记录，也是下一次挑战的起点。

## Layout

```
┌─────────────────────────────────────────┐
│                                         │
│            ┌───────────────┐            │
│            │   剑已断       │ ← 纯白 #FFFFFF, 36px
│            │               │            │
│            │     1250      │ ← 金墨 #D9A621, 48px
│            │      分       │ ← 纯白 #FFFFFF, 18px
│            │               │            │
│            │  [ 重新开始 ]  │ ← 墨色背景, 纯白文字
│            └───────────────┘            │
│                                         │
└─────────────────────────────────────────┘
```

- **Panel**: 400×300px, 居中, 墨色背景 (rgba(0.08, 0.08, 0.08, 0.92))
- **边框**: 2px, rgba(0.25, 0.25, 0.25, 0.5), 圆角 8px
- **标题**: "剑已断", 纯白, 36px, 居中
- **分数**: 数字金墨色 48px, "分" 纯白 18px
- **按钮**: 160×48px, 墨色背景, 纯白文字, 居中

## Animation

**墨迹侵蚀展开**:
- 时长: 0.5s
- 类型: Tween TRANS_CUBIC, EASE_OUT
- 效果: scale ZERO → ONE (从中心展开), alpha 0→1 (前 0.3s 完成)

## Interaction

- **重新开始按钮**: 点击 → hide_menu() → restore_hud() → GameStateManager.change_state(RESTART)
- **键盘**: 支持 Enter/Space 确认重新开始（Web 端无 gamepad 需求时）
- **自动聚焦**: 按钮在画面显示后自动获得焦点

## Responsive

- 使用 anchor PRESET_CENTER，480p-1440p 视口下不溢出
- 最小支持视口: 480×320 (移动端 Web)
- 文字不缩放，panel 尺寸固定

## Accessibility

- 纯白文字在深色背景上对比度 > 7:1 (WCAG AAA)
- 按钮有明确的 hover/pressed 状态反馈
- 支持键盘导航（Tab 聚焦按钮，Enter 激活）
