## 1. Visual Identity Statement

### One-Line Visual Rule

**"以留白衬笔触，以精准代华丽——每一帧都是一笔墨。"**

剑招是视觉核心，山水是画布。场景用极简水墨留白提供呼吸空间，三式流光在留白中书写战斗。没有多余的笔触——每一帧画面都像一幅已完成的水墨画，笔笔有力，不多不少。

### Supporting Visual Principles

#### Principle 1: 剑招即笔触 (Sword Techniques as Brushstrokes)
每一式剑招的流光轨迹就是一笔水墨——绕剑式的墨色圆弧、游剑式的金色细线、钻剑式的光锥聚爆。场景是宣纸，流光是墨。

*Design test*: 当犹豫"这个特效要不要加"时，问"这一笔是否让画面更像一幅完成的水墨画"——如果去掉它画面仍然完整，就不加。

*服务支柱*: 极简即美学 (Pillar 2)

#### Principle 2: 材质即画种 (Material as Medium)
不同的碰撞材质产生不同的"画种"——金属碰撞是金墨飞溅（火花），木杖碰撞是水墨渗裂（细纹），墨点炸碎是泼墨。材质反应不是特效，是水墨的不同表现形式。

*Design test*: 当犹豫"这个材质反应怎么做"时，参考真实水墨技法——泼墨、飞白、皴法——而不是数字粒子效果。

*服务支柱*: 精确即力量 (Pillar 1) — 材质反应让每一次精确命中都有独特视觉身份

#### Principle 3: 密度代替华丽 (Density over Decoration)
万剑归宗的壮观不是靠更多的颜色、更大的爆炸、更亮的光——而是靠同一种视觉语言的密度叠加。一百条同样的墨色弧线同时飞舞，比一百种不同颜色的粒子更壮观。

*Design test*: 当犹豫"万剑归宗够不够壮观"时，问"能不能只用这一种墨/金流光达到效果"——如果需要引入第三种视觉语言，说明方法错了。

*服务支柱*: 万剑归宗即高潮 (Pillar 5) + 极简即美学 (Pillar 2)

## 2. Mood & Atmosphere

核心思路：**用墨色密度和金色占比的变化来承载情绪，而不是靠色相变化。** 只有墨色+金色两个色相，靠光照强度、墨色密度、金色占比、笔触节奏、画面呼吸感来制造各状态的视觉差异。

### 状态 1：竞技场战斗

- **主要情绪**: 专注的紧张 — 不是狂热的厮杀，是画师落笔前屏息的那一瞬，持续整场战斗
- **光照**: 冷中性墨青底色，金色只在剑招轨迹出现时短暂暖化。光源从上方偏侧 45° 自然落下，如毛笔蘸墨时的反光
- **对比度**: 中高 — 前景角色与背景山水有明确分离
- **氛围形容词**: 沉着、锐利、克制、蓄势待发
- **能量级别**: 沉稳的高压 — 行草，流畅但每一笔都有控制力
- **视觉锚点**: 剑招轨迹作为画面中最亮的金色元素，在墨色中如金箔点缀

### 状态 2：万剑归宗高潮

- **主要情绪**: 狂喜的释放 — 所有积蓄的力量在一瞬间绽放，是整幅画完成最后一笔的酣畅
- **光照**: 暖金爆闪 — 金色从点缀变为主导，墨色被推到边缘。全向光源，从玩家向外辐射
- **对比度**: 极高 — 画面中心几乎白热化，边缘保留墨色轮廓
- **氛围形容词**: 辉煌、狂放、磅礴、目眩神迷
- **能量级别**: 狂热 — 草书中的狂草，笔走龙蛇，一气呵成
- **视觉锚点**: 万道金色流光汇聚又爆发，画面密度达到顶峰

### 状态 3：死亡 / 游戏结束

- **主要情绪**: 虚无的静止 — 不是戏剧性的死亡，是墨干了，笔停了
- **光照**: 彻底冷寂 — 墨色变为灰墨，金色完全消失。光源渐灭，如蜡烛熄灭后的余烬
- **对比度**: 低 — 画面趋近单色灰阶，如干涸的砚台
- **氛围形容词**: 空寂、枯涩、消散、万物归墨
- **能量级别**: 死寂 — 搁笔后的空桌
- **视觉锚点**: 玩家角色轮廓渐变为淡墨，最终融入背景山水，如画中人归于画

### 状态 4：波次间歇

- **主要情绪**: 平静的蓄势 — 暴风雨之间的宁静，但墨未干
- **光照**: 微暖 — 墨色为主，偶有淡金色微光飘过。月光般的侧光从画面一侧安静照入
- **对比度**: 低中 — 柔和的水墨晕染，边界模糊
- **氛围形容词**: 宁静、悠远、空灵、蓄势
- **能量级别**: 冥想 — 研墨时的专注
- **视觉锚点**: 远处山影如淡墨晕染，近处偶尔飘过一两滴金色墨点

### 状态 5：菜单 / HUD

- **主要情绪**: 肃穆的邀请 — 像一幅装裱好的水墨画，等待观者走入
- **光照**: 中性偏冷 — 墨色为骨，金线为饰。均匀正面柔光，如展厅照明
- **对比度**: 中 — 文字/选项的金色描边与墨底形成清晰可读对比
- **氛围形容词**: 典雅、端庄、留白、邀请
- **能量级别**: 静止但有引力 — 卷轴展开的那一瞬
- **视觉锚点**: 标题以金墨书法呈现，笔锋走势暗示游戏的核心动作

### 状态间转换

| 转换 | 视觉手段 | 时长 |
|------|---------|------|
| 战斗 → 万剑归宗 | 金色密度从轨迹向全屏递增 | 2-3 秒蓄力 |
| 万剑归宗 → 战斗 | 金色急速收缩回剑身，墨色回填 | 1 秒 |
| 战斗 → 死亡 | 金色瞬间熄灭，墨色渐灰，角色轮廓溶解 | 3-4 秒 |
| 死亡 → 间歇 | 灰墨重新晕开，微光出现，新一波从远景走来 | 2 秒 |
| 间歇 → 战斗 | 远景敌人轮廓变实，墨色加深，金色微光凝结为剑 | 1-2 秒 |
| 任意 → 菜单 | 全画面墨化为静态水墨画，UI 从墨中浮现 | 1 秒 |

## 3. Shape Language

核心原则：**剑招轨迹永远是画面中最亮的元素。** "以留白衬笔触"——留白（背景）衬托笔触（剑招），而不是反过来。

### 角色轮廓哲学

**方向：山水符号式** — 敌人轮廓基于山水元素（松、石、水、云、竹），与场景风格统一。

- 每个敌人类型以一种山水元素为剪影基础：松（坚韧型）、石（重甲型）、水（流动型）、云（远程型）、竹（敏捷型）
- 在缩略图大小下，靠形状主题（而非细节）区分敌人——松的挺拔、石的厚重、水的流畅各有可辨识的轮廓线
- 玩家角色的轮廓以"剑"为核心：人形简化为持剑者的最基本线条，剑是剪影中最突出的延伸

*服务支柱*: 三式皆平等 (Pillar 4) — 不同敌人形态迫使玩家使用不同剑式

### 环境几何风格

**方向：锐角为主** — 低多边形的本质是锐角，与中国山水画"斧劈皴"的硬朗笔触契合。

- 场景主体（山、石、建筑）保持锐角几何，明暗面分明，天然适合水墨风格的黑白对比
- 近处地面可用少量大面片平滑处理，但不追求曲线平滑
- 锐角几何在着色器中产生自然的面光差异，减少对额外光照的需求——Web 性能友好

*服务支柱*: 精确即力量 (Pillar 1) — 锐角几何的分明面光呼应"每一剑都有意义"的精确感

### UI 形状语法

**方向：墨迹侵蚀式** — UI 边缘像墨水在宣纸上自然晕开，是水墨世界的自然延伸而非叠加层。

- 血条表现为一滴墨在消耗（墨滴从饱满到干涸）
- 连击计数器表现为墨点在积累（密度随连击增加）
- HUD 读数用金色墨迹书写，与三式的金色流光视觉语言一致
- 战斗 HUD 保持几何规则边缘以确保快速读取（功能优先于装饰）

*服务支柱*: 极简即美学 (Pillar 2) — UI 的"华丽"来自墨迹密度变化，而非装饰

### 视觉层次（主角形状 vs 辅助形状）

| 层级 | 形状角色 | 具体元素 | 视觉权重 |
|------|---------|---------|---------|
| **主角** | 剑招轨迹 + 玩家角色 | 绕剑式圆弧、游剑式细线、钻剑式光锥 | 最亮（金色），最高对比 |
| **次级** | 敌人轮廓 | 山水符号式剪影 | 中等（墨色轮廓），次高对比 |
| **辅助** | 环境地形 | 锐角几何山石 | 低（淡墨），低对比 |
| **背景** | 远景山水 | 极简墨色晕染 | 最低（近乎留白），最低对比 |

万剑归宗时，主角形状扩展为"全屏金色轨迹"——密度代替华丽，但仍是同一种形状语言（线条），只是数量暴增。

## 4. Color System

核心原则：**统一色板——UI 直接使用墨色/金色，与世界完全统一。** 这符合"极简即美学"支柱和"墨迹侵蚀式"UI 风格。

### 主色调板

| 颜色 | Hex 参考 | 角色 | 世界含义 |
|------|---------|------|---------|
| **墨黑** | #1A1A2E | 场景底色、绕剑式流光、敌人轮廓 | 水墨世界的基底，一切从墨中来 |
| **宣纸白** | #F0EDE5 | 留白空间、远景淡墨、波次间歇微光 | 呼吸空间，水墨画的"气" |
| **金墨** | #D4A843 | 游剑式剑气、钻剑式粒子、UI 读数 | 剑道的力量与精确，锋芒毕露 |
| **金白** | #F5E6B8 | 万剑归宗爆发、高密度粒子聚爆 | 力量的极致——金墨的白热化 |
| **淡墨灰** | #4A4A5E | 环境地形、近处山石、次要元素 | 远离焦点的退隐——服务于主角形状 |
| **灰墨** | #2E2E3E | 死亡状态色调、干涸感 | 墨已干枯——虚无与结束 |
| **墨青** | #1E2D3D | 竞技场战斗底色、冷色温基准 | 沉着、冷静的战斗基调 |

### 语义色彩词汇表

| 语义 | 颜色 | 为什么是这个颜色 |
|------|------|-----------------|
| **力量 / 攻击** | 金墨 (#D4A843) | 剑气的金色是力量的可视化——玩家的每一次出招都是金色的 |
| **防御 / 化解** | 墨黑 (#1A1A2E) | 绕剑式的墨色流光代表以柔克刚——用黑暗化解光明的攻击 |
| **危险 / 受伤** | 墨黑加深 + 金墨减弱 | 不使用红色——红色在这个水墨世界中不存在。危险用"墨色吞噬金色"来表达 |
| **高潮 / 爆发** | 金白 (#F5E6B8) | 万剑归宗的极致——金色被推到白热化，是力量的最高形态 |
| **安全 / 间歇** | 宣纸白 (#F0EDE5) | 留白是安全的——宣纸上的空白是未被墨染的地方 |
| **结束 / 死亡** | 灰墨 (#2E2E3E) | 墨已干枯——一切色彩消失，只余灰烬 |
| **计数 / 数据** | 金墨 (#D4A843) | UI 读数用金墨书写——数据也是剑道的一部分 |

### 色温规则（按状态）

| 游戏状态 | 主色温 | 金色占比 | 墨色占比 |
|---------|--------|---------|---------|
| 竞技场战斗 | 冷（墨青底） | 低（15-20%，仅剑招轨迹） | 高（80-85%） |
| 万剑归宗 | 暖（金白爆闪） | 极高（60-80%） | 低（边缘保留） |
| 死亡 | 冷寂（灰墨） | 0% | 100%（灰阶） |
| 波次间歇 | 微暖（宣纸底） | 极低（5%，微光点缀） | 中（40%）+ 留白（55%） |
| 菜单/HUD | 中性偏冷 | 中（30%，文字/描边） | 中（40%）+ 留白（30%） |

### UI 色板

UI 使用统一色板（与世界一致），但有明确的层级规则：

| UI 元素 | 颜色 | 说明 |
|---------|------|------|
| HUD 文字/数字 | 金墨 (#D4A843) | 与剑招流光统一 |
| HUD 背景/容器 | 墨黑 (#1A1A2E) 半透明 | 墨迹侵蚀式边缘 |
| 生命值（满） | 金墨 (#D4A843) | 力量的可视化 |
| 生命值（低） | 灰墨 (#2E2E3E) | 墨滴在消耗、干涸 |
| 连击计数 | 金墨密度递增 | 墨点在积累 |
| 菜单选项（未选中） | 淡墨灰 (#4A4A5E) | 退隐 |
| 菜单选项（选中） | 金墨 (#D4A843) | 聚焦 |
| 菜单边框/分隔 | 墨色笔触 | 墨迹侵蚀式 |

### 色盲安全

本游戏的核心配色是**墨色+金色**（深蓝黑+暖黄），这本身是一个色盲安全的组合：

| 语义 | 颜色 | 色盲风险 | 备份方案 |
|------|------|---------|---------|
| 三式区分 | 墨黑 vs 金色 | 低风险——明度差异大（墨黑极暗 vs 金色明亮） | 每式有独特的轨迹形状（圆弧/细线/光锥）作为形状备份 |
| 生命值 | 金墨→灰墨 | 低风险——用填充量变化而非颜色变化 | 墨滴大小（满=大滴，空=小滴）作为形状备份 |
| 危险指示 | 墨色吞噬金色 | 中风险——暗+暗可能难以辨认 | 屏幕边缘墨迹侵蚀动画作为运动备份 |
| 万剑归宗触发 | 金白爆发 | 无风险——白热化是最高亮度 | 声音反馈（爆发音效）作为听觉备份 |

**结论**：由于本游戏不使用红/绿语义配色，色盲风险整体较低。三式通过**形状**（圆弧/细线/光锥）区分而非颜色，这已是天然的色盲安全设计。

## 5. Character Design Direction

核心原则：**角色是剑招的容器，不是视觉的主角。** 角色简化是为了让剑招轨迹更清晰。

### 玩家角色

**视觉原型：水墨风格化的持剑者** — 极简人体比例，关节简化，重点在剑的延伸和流光轨迹。

**从学徒到剑圣的视觉进化：纯动态表现**
- 不依赖等级或属性——视觉进化完全由玩家在当前局的表现驱动
- **学徒态**（低连击/未触发万剑归宗）：角色以墨色为主，流光稀疏，剑招轨迹短
- **渐进态**（连击积累中）：金墨纹饰在角色表面逐渐浮现，如墨迹中的金箔透出
- **剑圣态**（触发万剑归宗后）：金墨占比大幅增加，流光密度最高，角色轮廓被金色轨迹包围
- 每局结束后角色回归墨色基态——视觉进步与游戏进度同步，局外不留存

*服务支柱*: 玩家即进度 (Pillar 3) — 视觉表达进步，但不提供数值优势

### 敌人类型（5 种）

| 敌人类型 | 山水元素 | 剪影特征 | 推荐剑式 | 视觉辨识 |
|---------|---------|---------|---------|---------|
| **松韧型** | 松 | 挺拔竖直，枝干般延伸 | 钻剑式（穿透松节） | 细长竖向轮廓 |
| **重甲型** | 石 | 厚重方正，底部宽稳 | 绕剑式（化解硬力） | 宽大横向轮廓 |
| **流动型** | 水 | 流线弯曲，无固定形状 | 游剑式（缠绕附着） | 曲线波浪轮廓 |
| **远程型** | 云 | 虚浮上飘，体积感弱 | 钻剑式（穿透虚体） | 轻盈圆形轮廓 |
| **敏捷型** | 竹 | 细长尖锐，重心偏高 | 绕剑式+游剑式 | 尖锐三角轮廓 |

**区分规则**：在缩略图大小下，靠**形状主题**（竖/方/曲/圆/三角）区分，不靠颜色或细节。每种敌人用同一种墨色，仅在轮廓形状上不同。

### 表情与姿态风格

**方向：水墨风格化** — 高度风格化，关节简化，参考水墨画中的人物比例。

- 角色不做面部表情——水墨画中的人物从来不需要表情来传达意图
- 姿态通过**剑的方向和身体倾斜角度**传达意图——攻击时前倾，防御时后仰
- 动画节奏：非匀速——出招瞬间快（如笔锋转折），收招慢（如墨迹收干），符合水墨画的"疾涩"节奏

### LOD 哲学

**方向：两级 LOD + 水墨降级**

| 距离 | 渲染方式 | 三角面预算 |
|------|---------|-----------|
| 近距离（玩家+近距敌人） | 完整低多边形模型 + 水墨着色器 | 每角色 < 500 面 |
| 远距离（远景敌人/环境） | 水墨晕染色块代替模型 | 0 面（纯着色器/贴图） |

- 远景敌人直接渲染为墨色剪影色块——不做模型 LOD，直接替换为 2D 水墨画
- 玩家角色始终保持最高精度——这是画面的锚点
- Web 性能降级：当帧率下降时，首先降远景为纯色块，其次减少近距敌人面数

## 6. Environment Design Language

核心原则：**场景是宣纸——它的唯一职责是让剑招的笔触更清晰。** 场景不抢戏。

### 建筑 / 场景风格

**方向：山水画意象** — 不做写实建筑，而是用水墨山水画的构图原则构建竞技场。

- 竞技场本身没有明确的建筑结构——它是山水画中的一个"场景切片"
- 地形元素（山石、平台、柱、水池）以水墨画中的典型元素出现：斧劈皴的岩石、披麻皴的土坡
- 不做围墙或边界——用远景的水墨晕染暗示空间边缘，如画卷的留白边界
- 参考牧神记动画中的场景处理：环境是流动的，有生命的，但始终服务于前景动作

### 贴图哲学

**方向：手绘水墨质感** — 不做 PBR，不用真实材质。所有表面用手绘水墨笔触表现。

- 地面：宣纸质感 + 淡墨晕染，偶尔有干笔飞白的纹理
- 岩石/山体：斧劈皴笔触直接画在模型上，明暗面分明
- 水面：留白 + 淡墨倒影，不做真实水面反射
- 为什么不做 PBR：PBR 追求真实感，而这个游戏追求的是"画感"——水墨画从来不是真实的，它是真实的提炼

### 道具密度规则

**方向：极简留白为主** — 道具越少，剑招越突出。

| 区域类型 | 道具密度 | 规则 |
|---------|---------|------|
| 战斗核心区 | 极低 | 只有地面和必要的地形起伏——留白是主角 |
| 地形边缘 | 低 | 偶尔的岩石/竹枝作为绕剑式附着点 |
| 远景 | 无道具 | 纯水墨晕染色块——不渲染任何实体 |

- 道具的唯一功能性角色：为绕剑式提供附着表面（岩石、竹竿、地面）
- 装饰性道具仅出现在玩家不可能到达的远景位置

### 环境叙事指南

不做文字叙事，用视觉细节讲述世界：

| 细节 | 叙事含义 |
|------|---------|
| 地面的剑痕 | 这个竞技场被无数剑客使用过——你不是第一个，也不会是最后一个 |
| 断裂的竹 | 之前的战斗留下了痕迹——战斗的破坏力是真实的 |
| 岩石上的墨迹 | 某种力量（剑气？）侵蚀了环境——暗示万剑归宗的历史 |
| 远景的山水轮廓 | 这个世界比竞技场更大——但你现在只关注眼前 |

### 两个区域的视觉差异化

| 特征 | 区域 1：山石 | 区域 2：水竹 |
|------|------------|------------|
| **主导几何** | 锐角（斧劈皴） | 半曲线（披麻皴） |
| **主色** | 墨青 + 淡墨灰 | 墨黑 + 宣纸白 |
| **地面质感** | 粗糙岩石面，飞白纹理 | 光滑水面/泥地，晕染过渡 |
| **垂直元素** | 尖锐岩石柱 | 柔韧竹林 |
| **氛围** | 沉重、险峻、压迫 | 空灵、流动、飘逸 |
| **出现敌人** | 松韧型、重甲型、远程型 | 流动型、敏捷型 |
| **光照** | 强烈侧光，面光分明 | 柔和散射光，边界模糊 |
| **着色器效果** | 面光差异大，黑白对比强 | 渐变过渡多，墨色层次丰富 |

**设计测试**：如果玩家闭上眼睛再睁开，能立刻分辨自己在哪个区域——差异化就成功了。

## 7. UI/HUD Visual Direction

核心原则：**UI 是墨迹世界的自然延伸，不是叠加层。** 像水墨画上的题款和印章——属于画面，但有独立身份。

### 视觉风格

**方向：墨迹侵蚀式** — UI 边缘像墨水在宣纸上自然晕开，不规则边界。

- HUD 容器用墨色半透明背景，边缘做墨迹侵蚀效果（不规则、有渗透感）
- 但战斗 HUD 的数据区域保持几何规则边缘——功能优先于装饰，确保快速读取
- 菜单/UI 面板用更明显的墨迹边缘——菜单不紧急，可以更风格化

### 排版方向

- **字体风格**：金墨书法体——标题用行书/草书风格，正文用楷书风格
- **字重层级**：标题粗（金墨实心），正文中（金墨半透明），说明文字细（淡墨灰）
- **大小层级**：标题 > 数据读数 > 提示文字 > 版权信息
- **可读性**：所有文字必须在墨色背景上有足够对比度——金墨在墨黑上永远清晰

### HUD 战斗界面

| HUD 元素 | 视觉表现 | 读取方式 |
|---------|---------|---------|
| 生命值 | 一滴墨在消耗——满血=饱满墨滴，低血=干涸墨迹 | 墨滴大小（非颜色变化） |
| 连击计数 | 墨点在积累——连击越高墨点密度越大 | 墨点数量 + 金色数字 |
| 当前剑式指示 | 三式图标在 HUD 底部——当前激活的用金墨，未激活的用淡墨 | 图标亮度 |
| 万剑归宗蓄力 | 墨色圆环逐渐填满金色——填满时触发 | 圆环完成度 |
| 波次计数 | 金色数字 + 墨色笔触装饰 | 数字 |

### 菜单设计

- **标题画面**：游戏名以金墨书法呈现，背景是极简水墨山水。笔锋走势暗示剑招动作
- **暂停菜单**：半透明墨色覆盖，选项用金墨文字，选中项有墨迹扩散动画
- **游戏结束**：画面墨化为静态水墨画，"再来一次"从墨中浮现——呼应死亡→间歇的转换
- **得分画面**：数据用金墨书写在宣纸质感背景上——像一幅记录战果的书法

### UI 动画风格

- **出现**：墨滴落纸般扩散（300-500ms）
- **消失**：墨迹收干般淡出（200-300ms）
- **选中反馈**：金墨从中心向外辐射（150ms）
- **数据变化**：数字有轻微的墨迹抖动（100ms）——像刚写下的墨迹未干

### 可读性保障

| 场景 | 潜在问题 | 解决方案 |
|------|---------|---------|
| 万剑归宗（全屏金色） | HUD 文字可能被金色淹没 | 万剑归宗时 HUD 淡出，结束后淡入 |
| 死亡（灰阶画面） | 金墨文字在灰底上可能对比度不足 | 死亡菜单文字用纯白而非金墨 |
| 战斗密集时 | HUD 分散注意力 | 3 秒无受伤后 HUD 自动半透明化 |

## 8. Asset Standards

核心原则：**Web 性能是硬约束，不是软目标。** 所有资产标准必须满足 HTML5 导出的性能预算。

### 性能预算（来自 technical-preferences.md）

| 预算项 | 限制 | 对资产的影响 |
|--------|------|-------------|
| Draw call | < 50/帧 | 合并网格、共享材质、Atlas 纹理 |
| 三角面 | < 10K 总计 | 低多边形模型，两级 LOD |
| 同时活跃敌人 | < 10 | 每个敌人 < 500 面 |
| 同时活跃粒子 | < 50 | 流光轨迹用 LineRenderer 而非粒子系统 |
| 后处理 pass | < 2 | 最多一个描边 pass + 一个色调 pass |
| ZIP 包大小 | < 50MB gzipped | 纹理压缩、音频压缩、模型优化 |

### 模型规范

| 资产类型 | 面数预算 | LOD 规则 |
|---------|---------|---------|
| 玩家角色 | < 500 面 | 始终最高精度，不做 LOD |
| 近距敌人 | < 500 面/个 | 近距离完整模型 |
| 远距敌人 | 0 面 | 用 2D 水墨剪影色块代替 |
| 地形模块 | < 200 面/块 | 模块化拼接，共享材质 |
| 道具（岩石/竹） | < 100 面/个 | 极简几何，锐角为主 |
| 竞技场总计 | < 3K 面 | 留 7K 面给角色和特效 |

### 纹理规范

| 规格 | 标准 | 说明 |
|------|------|------|
| 格式 | WebP / ASTC | Web 友好的压缩格式 |
| 最大分辨率 | 512x512 | 低多边形不需要高分辨率纹理 |
| 色彩空间 | sRGB | Web 标准 |
| 手绘风格 | 必须 | 不做 PBR 贴图，所有纹理手绘水墨风格 |
| Atlas 共享 | 推荐 | 多个角色/道具共享同一 Atlas 减少 draw call |

### 材质规范

| 规格 | 标准 | 说明 |
|------|------|------|
| 着色器语言 | Godot Shader Language (.gdshader) | 不用 GDExtension |
| 材质数量 | < 15 场景总计 | 共享材质减少 draw call |
| 着色器复杂度 | 低 | WebGL 2.0 兼容，避免 compute shader |
| 后处理 | < 2 pass | 描边 + 色调调整 |

### 流光轨迹规范

流光轨迹是本游戏最核心的视觉资产：

| 元素 | 实现方式 | 性能考量 |
|------|---------|---------|
| 绕剑式墨色轨迹 | LineRenderer + 墨色材质 | 比粒子系统省 90% 性能 |
| 游剑式金色细线 | LineRenderer + 金色材质 | 同上 |
| 钻剑式光锥 | 简单 MeshInstance + 缩放动画 | 1 个三角锥体 |
| 万剑归宗流光 | LineRenderer 批量渲染 | 同一材质多条线，1 个 draw call |
| 墨点炸碎 | < 10 个 Sprite3D | 预设动画，非实时粒子 |
| 金属火花 | < 5 个 Sprite3D | 预设动画 |
| 木杖裂纹 | Decal + 裂纹贴图 | 1 个 draw call |

### 音频规范

| 规格 | 标准 | 说明 |
|------|------|------|
| 格式 | OGG Vorbis | Web 友好，体积小 |
| 采样率 | 22050 Hz | 足够 Web 游戏质量 |
| 音效数量 | < 30 个 | 精简音效库 |
| 音乐 | 循环片段 | 氛围音乐用短循环，不做长曲目 |

### 文件命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 模型 | `mdl_[name].glb` | `mdl_player.glb`, `mdl_enemy_pine.glb` |
| 纹理 | `tex_[name].webp` | `tex_ink_splash.webp` |
| 着色器 | `shd_[name].gdshader` | `shd_ink_trail.gdshader` |
| 材质 | `mat_[name].tres` | `mat_gold_qi.tres` |
| 场景 | `[Name].tscn` | `ArenaMountain.tscn`, `Player.tscn` |
| 音效 | `sfx_[name].ogg` | `sfx_sword_hit_metal.ogg` |
| 动画 | `anim_[name].tres` | `anim_player_slash.tres` |

### Web 导出特别约束

- **禁止 GDExtension**：所有代码和着色器必须用 GDScript/Godot Shader Language
- **纹理压缩**：导出时自动压缩，但源文件保留高质量版本
- **音频流式加载**：大文件（音乐）使用流式加载而非预加载
- **场景预加载**：核心场景在启动时预加载，区域场景按需加载

## 9. Reference Direction

核心原则：**参考是起点，不是终点。** 每个参考源指定要取什么、要避开什么。

### 参考 1：牧神记动画

- **取什么**: 水墨风环境的氛围感——山石的笔触处理、光影的水墨渐变、场景的"活"感。传统武侠美学与现代 3D 渲染的融合方式。
- **避开什么**: 牧神记的角色设计偏复杂华丽——《归宗》的角色应更极简。不做牧神记级别的角色细节。
- **为什么重要**: 用户指定的首要视觉参考。验证了水墨 3D 风格在现代动画中的可行性。

### 参考 2：对马岛之鬼 — 黑白模式 (Kurosawa Mode)

- **取什么**: 黑白滤镜下的对比度控制——哪些元素保留、哪些元素消失。风、草、尘的动态在黑白中的表现方式。
- **避开什么**: 对马岛的写实光照——《归宗》追求的是"画感"而非"真实感"。不模仿其真实的光影物理。
- **为什么重要**: 证明了黑白/水墨风格在动作游戏中的可玩性——玩家不会因为缺少色彩而失去信息。

### 参考 3：Journey (风之旅人)

- **取什么**: 极简 3D 的情感表达——用最少的视觉元素传达最深的情感。沙丘的几何简化方式、流光轨迹的视觉语言、留白的力量。
- **避开什么**: Journey 的缓慢节奏——《归宗》是快节奏动作游戏。不模仿其冥想式步调。
- **为什么重要**: 证明了"少即是多"在 3D 游戏中的商业和艺术可行性。

### 参考 4：Hades

- **取什么**: 战斗反馈的"汁液感"——顿帧、屏幕震动、命中音效的精确配合。连击系统的视觉奖励机制。
- **避开什么**: Hades 的色彩爆炸和粒子密集——《归宗》用墨色+金色的克制代替 Hades 的多彩。
- **为什么重要**: 验证了动作 Roguelite 的核心循环设计——"再来一次"的动力机制。

### 参考 5：中国武侠电影（张艺谋《英雄》《十面埋伏》）

- **取什么**: 剑术编排的节奏感——出招的速度变化、蓄力的停顿、收招的优雅。色彩的象征性使用（红色=激情，蓝色=冷静）。
- **避开什么**: 电影的叙事驱动——《归宗》不做叙事。只取动作编排的节奏和姿态语言。
- **为什么重要**: 剑术动作的最终参考来源——游戏中的每一招都应该"像电影中的一幕"。

### 参考总结

| 参考 | 取什么 | 避开什么 |
|------|--------|---------|
| 牧神记动画 | 水墨环境氛围、3D 水墨融合 | 复杂角色设计 |
| 对马岛之鬼 (Kurosawa) | 黑白对比度控制、动态元素 | 写实光照 |
| Journey | 极简情感表达、留白力量 | 慢节奏冥想 |
| Hades | 战斗反馈"汁液感"、连击奖励 | 色彩爆炸、粒子密集 |
| 武侠电影 | 剑术节奏、蓄力/收招优雅 | 叙事驱动 |
